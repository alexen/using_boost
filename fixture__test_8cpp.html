Для "внешней" фикстуры констрктор и деструктор (как и setup() и teardown()) вызываются только один раз - конструктор перед первым тестом, деструктор - после последнего.

Локальная фикстура создается и разрушается (c-tor/d-tor) для каждого тест-кейса, поэтому в каждом тесте:

  • кол-во объектов всегда будет равно единице;
  • кол-во вызовов конструкторов и деструкторов будет увеличиваться.

Глобальная фикстура создается при старте программы, а затем, для каждого тест кейса все равно создается и уничтожается отдельная копия оъекта. Поэтому для глобальной фикстуры счетчик объектов будет равен двойке.
